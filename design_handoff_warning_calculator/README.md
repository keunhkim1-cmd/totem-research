# Handoff: 투자경고 계산기 재디자인 (Terminal × 모노크롬)

> 기존 `shamanism-research/index.html` 의 **투자경고 계산기** 탭을 Bloomberg × TradingView × Toss 베이스의 흑백 모노크롬 터미널 톤으로 재디자인한 결과물의 핸드오프 문서입니다.

---

## Overview

이 핸드오프는 기존 샤머니즘 리서치 프로젝트(`index.html` 단일 파일)의 "투자경고 계산기" 페이지를 더 밀도 높은 프로용 리서치 툴 톤으로 바꾸는 작업을 위한 것입니다. 기존 API·데이터 파이프라인(`/api/*.py`, `/lib/*.py`)은 그대로 두고 **프론트엔드 레이아웃·타이포·색만 교체**하는 것이 목표입니다.

핵심 변경:

- Apple dark SF-style → **Bloomberg-grid 흑백 터미널**
- 파란 accent(`#0071e3`) → **흰색/오프화이트 단일 accent** (Mono · Night)
- 21pt 카드 기반 레이아웃 → **밀도 높은 섹션/테이블/좌측 바 헤더** 구조
- 결과 카드 4~5개로 분산 → **타임라인 → 조건 테이블 → Verdict 카드 → 차트 → 기본 시세** 단일 스트립
- 원본의 투자금지 밈 티커 테이프(⛩ TABOO) **유지**

## About the Design Files

**이 번들의 파일은 디자인 레퍼런스입니다. 프로덕션 코드로 그대로 복사하지 마세요.**

- `reference/Redesign Options.html` 와 그 안의 `mockups/*.jsx` 는 HTML + React(Babel in-browser)로 짜인 **의도를 보여주는 목업**입니다.
- 실제 포팅 대상은 기존 `shamanism-research/index.html` (vanilla JS · 단일 파일) 입니다. 기존 구조(페이지 전환, 검색 플로우, API 호출, 전역 상태)를 유지하면서 **CSS 토큰·컴포넌트 마크업만 이 디자인 톤으로 교체**하세요.
- 목업에 쓰인 `IBM Plex Mono / IBM Plex Sans` 는 **무시**하세요. 원본 `index.html` 이 이미 쓰고 있는 `SF Pro Display / SF Pro Text / SF Mono` 를 **그대로 유지**합니다. (이 핸드오프의 모든 문서에서 폰트는 SF Pro 체계로 기술되어 있습니다.)
- 목업의 mock 데이터(`mockups/mock-data.jsx`) 는 에코프로 스냅샷입니다. 실제 바인딩은 기존 `/api/caution-search`, `/api/warn-search`, `/api/stock-price`, `/api/stock-overview` 응답을 사용하세요.

## Fidelity

**High-fidelity (hifi)**. 색상·간격·타이포·섹션 구조·애니메이션 모두 최종안으로 확정된 상태입니다.

단, 아래 두 항목은 **의도적으로 느슨**하게 두었습니다:

1. **차트 렌더러** — 목업은 손으로 그린 SVG 패스입니다. 실제 구현은 NAVER fchart 응답을 받아 Recharts / ECharts / Lightweight-Charts / 또는 기존 프로젝트가 쓰는 차트 라이브러리로 그리세요. 스타일 규칙(1.5px stroke, 단색 fill gradient, 3본의 dashed threshold line)만 맞추면 됩니다.
2. **사이드바 관심종목 더미** — 본 구현에선 사용자의 실제 관심종목 저장소가 없으므로 **섹션 자체를 제거하거나 "(개발중)" 상태로** 두세요. Mock의 6종목은 단지 레이아웃 참고입니다.

## Interactions & Behavior

현재 시안은 **정적 상태 1개(조건 ①② 유지 / ③ 이탈 = 다음 거래일 해제)**만 보여줍니다. 실제 구현 시 다음 상태 전이를 처리해야 합니다:

### 1. 검색 → 결과 표시

기존 `index.html` 의 `onClick="stockSearch()"` 플로우를 그대로 유지. 차이점은 **결과를 보여주는 컨테이너가 카드 4~5개에서 단일 섹션 스택으로 바뀜**.

- 종목 선택 → API 호출 → **Symbol Header**부터 순서대로 채움
- 빈 상태(종목 미선택): "종목을 먼저 검색하세요" placeholder. 섹션 헤더만 남기고 내용 영역은 `color: var(--text-mute)` 로 dimmed.

### 2. 조건 테이블 상태 (`HOLD` / `CLEAR` / `N/A`)

각 행에 3가지 상태:

| 상태 | 조건 | 플래그 색 | 행 hover |
|---|---|---|---|
| `유지` (HOLD) | T 종가 ≥ 임계가 | 빨강 (`--up` `#F04452`) | 옅은 빨강 배경 |
| `이탈` (CLEAR) | T 종가 < 임계가 | 초록 (`--ok` `#4ADE80`) | 옅은 초록 배경 |
| `—` (N/A) | 비가격 사유(관리종목 등) | 회색 | 없음 |

3 조건이 모두 계산되지 않았을 때(장 미마감, 거래정지 등)는 전체 테이블을 **skeleton 회색 바**로 대체. Verdict 카드도 "판정 보류" 토스트.

### 3. Verdict 카드 결과

세 가지 변형:

- **다음 거래일 해제** (조건 중 하나라도 CLEAR) → 아이콘 ✓ · 배경 그라디언트 `--ok-soft` → transparent · `.tag` = `다음 거래일 해제`
- **경고 유지** (3 조건 모두 HOLD) → 아이콘 ⚠ · 배경 `--amber-soft` · `.tag` = `경고 유지`
- **투자위험 상향 경고** (조건 재충족·특수) → 아이콘 ▲ · 배경 `--up-soft` · `.tag` = `위험 상향 가능`

### 4. 타임라인 셀

`D0 → T+1 ~ T+9 → 해제` 총 11셀. 상태별 색:

- `past` (경과) : 초록 (`--ok-soft` 배경)
- `today` (현재 거래일 T) : 앰버(흰색) 배경 · 검정 글자
- `release` (해제 예정일) : 파랑 (`--toss-soft` 배경)

**동작**: 장 마감 전이면 `today` 셀이 `T+n` (현재 회차)에 고정. 장 마감 후 다음 거래일이 될 때까지 `today` 셀 그대로.

### 5. 차트 탭 (5D / 15D / 30D)

기본 `30D` 선택. 탭 클릭 시 기간만 바뀌고, 3개 threshold overlay 라인은 항상 유지.

### 6. 티커 테이프

`60s linear infinite` 가로 스크롤. Hover 시 `animation-play-state: paused`. 문구는 기존 `index.html` 의 금지어 배열 그대로.

### 7. 네비게이션

상단 탭 4개 — 기존 `switchPage()` 핸들러 그대로. 개발중인 탭(`재무 현황`, `공시 분석기`)은 라벨에 `(개발중)` 서픽스를 붙이고 클릭 시 원본 페이지로 이동. 이 시안은 **`경고계산기` 탭 내부 화면만** 다룹니다.

## State Management

기존 `index.html` 에 이미 있는 전역 상태를 그대로 사용:

```js
// 현재 선택된 종목
window.currentStock = { name, code, level, designationDate, ... }
// 계산 결과
window.warningResult = { conditions: [...], verdict, releaseDate, ... }
// 기준가 현황
window.priceSnapshot = { tOpen, tClose, dayHigh, dayLow, volume, ... }
// 관심종목 (localStorage)
window.watchlist = [{ name, code, dday, ... }]
```

**새로 추가할 상태 없음**. 기존 응답을 이 레이아웃에 재배치하는 작업입니다.

- **폰트**: **기존 `index.html` 의 SF Pro Display / SF Pro Text / SF Mono 그대로**
- **간격 단위**: 4 / 6 / 8 / 10 / 14 / 16 / 24 (px)
- **보더 radius**: 0 (기본) / 8 (기존 카드 호환 모드)
