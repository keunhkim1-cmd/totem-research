# TONE · 톤 가이드

이 시안을 한 줄로: **"프로용 리서치 터미널의 밀도 × 모노크롬 세련됨 × 원본 샤머니즘 리서치의 장난스러운 내부 농담"**.

타깃 유저는 전업 투자자·PB·애널리스트. 친근함보다 **"아는 사람 눈에만 보이는 정확성"** 이 우선.

---

## 🎯 디자인 원칙 (우선순위 순)

### 1. 정보 밀도가 카피 친절함보다 중요하다

- 한 화면에 **한 질문의 답**: "내일 해제될까?" 그 질문의 답(타임라인·조건·Verdict)이 상단 60%에.
- **PER · PBR · 시총 · 52주 고저** 같은 "일반적 기업 정보"는 이 화면에서 2차 정보. 숨기거나 후순위로.
- 카드 안에 카드(중첩 박스) 금지. 섹션은 수평선(`1px hairline`)으로만 구분.

### 2. 색은 단조롭게, 신호는 날카롭게

- **회색 3단계 + 흰색**이 본문 전부.
- 색은 **신호로만**:
  - 🔴 `--up #F04452` — 상승 / 경고 유지 / 조건 충족 (한국식 빨강 상승)
  - 🔵 `--dn #3485FF` — 하락 / 단순 하락만
  - 🟢 `--ok #4ADE80` — 해제·이탈·긍정 Verdict
  - 🔷 `--toss #3182F6` — D-day · 정보 액센트
  - ⚪ `--amber #FFFFFF` — **포커스·섹션 좌측 바 (Mono Night에서는 순백)**
- 그라디언트는 **Verdict 카드 배경 1곳**에서만. 그 외 금지.

### 3. 타이포는 "SF Pro 2종 + 숫자는 반드시 등간격"

기존 `index.html` 의 폰트 시스템을 **그대로 유지**:

```css
--font-display: 'SF Pro Display', 'Helvetica Neue', 'Helvetica', 'Arial', sans-serif;
--font-text:    'SF Pro Text',    'Helvetica Neue', 'Helvetica', 'Arial', sans-serif;
--mono:         'SF Mono', 'Menlo', 'Consolas', monospace;
```

사용 규칙:

| 요소 | 폰트 | 크기·굵기 |
|---|---|---|
| 대형 가격·숫자 | `--mono` (또는 `--font-text` + `font-variant-numeric: tabular-nums`) | 22pt / 500 |
| 테이블 숫자 셀 | `--mono` | 11pt / 400 |
| 섹션 헤더 | `--font-display` | 11pt / 500 / **uppercase** / `letter-spacing: 1.5px` |
| 본문 카피 (Verdict 등) | `--font-text` | 12~15pt / 400~500 |
| 라벨·출처·hint | `--font-text` | 9~10pt / 500 / **uppercase** / `letter-spacing: 0.8px` |
| 탭·버튼 | `--font-display` | 10pt / 500 / **uppercase** |

**숫자는 반드시 `font-variant-numeric: tabular-nums`**. 등간격이 아니면 테이블이 망가짐.

### 4. 한글/영어 혼용 규칙

- **기본 카피 = 한글**. 원본 `index.html` 의 톤을 계승.
- **유지할 영문**: PER · PBR · KRX · KOSDAQ · DART · NAVER · T · T-5 · T-15 · D0 · T+n · AND · Δ · 12.7%
- **한글화할 것**: HOLD/CLEAR → 유지/이탈 · REL → 해제 · RELEASE NEXT DAY → 다음 거래일 해제 · SRC: → 출처: · RULE: → 규정:
- 숫자 구분자는 **콤마** (`toLocaleString('ko-KR')`). 원 단위는 **생략** (`328,000` 식).

### 5. 섹션 헤더는 항상 동일 리듬

```
[왼쪽 2px 흰바][ 섹션명(대문자·한글 가능) ]      [우측: 출처/부연(9pt, mute)]
```

이 패턴이 **타임라인 · 조건 · 차트 · 기본 시세 · KRX 규정 · 관심종목 · 공시 피드** 7개 섹션 전부에 일관되게 적용. 예외 금지.

## ✅ DO

- **여백은 수직으로는 작게(11 · 14 · 16px), 좌우로는 중요 (16px padding).**
- **숫자는 오른쪽 정렬**, 라벨은 왼쪽 정렬. 테이블의 모든 th/td 규칙.
- **Δ · % · × 같은 기호는 숫자 앞 또는 뒤에 붙여** 별도 cell로 쪼개지 말 것 (e.g. `+45.00%` 한 셀, `1.45×` 한 셀).
- **↑↓ 화살표는 ▲▼** (유니코드). SVG 아이콘 말 것.
- **시계·D-day 등 메타정보는 상단 오른쪽 코너에 9~10pt로**.
- **Verdict 카드 1곳만 warm 톤**. 나머지는 차가운 터미널 톤을 깬 예외가 없어야 함.
- **티커 테이프 ⛩ prefix는 유지**. 원본의 샤머니즘 DNA.

## ❌ DON'T

- **그라디언트 배경 금지** (Verdict 카드 한 곳 제외).
- **rounded corner `border-radius: 8px+` 금지**. 기본값 0. 원본 `index.html` 의 둥근 카드와는 의도적으로 다름.
- **drop-shadow·glow 금지**. 터미널은 flat.
- **아이콘 · 이모지 남용 금지**. 티커의 ⛩ 외엔 Verdict 카드의 SVG 체크 아이콘 1개만.
- **"AI / 인사이트 / 스마트" 같은 부가적 카피 금지**. 라벨은 명사구, 서술형 없음.
- **폰트 3종 이상 금지**. SF Pro Display + SF Pro Text + SF Mono 3종이 끝.
- **애니메이션 남용 금지**. 허용: 티커 테이프 스크롤 · Verdict 카드 페이드인 · 네비 밑줄 트랜지션. 그 외 금지.
- **"투자 권유"로 해석될 수 있는 단정 카피 금지**. Verdict 카드의 긍정 톤에는 "KRX §4-2 규정상" 같은 근거 문구 필수.

## 🚨 드리프트 방지

개발 중 "조금 더 친절하게 / 색 하나 더 / 카드 둥글게" 충동이 올 수 있습니다. 그때마다:

1. **이 페이지는 1초 안에 "해제 여부"를 읽어내야 한다** — 장식은 시간을 뺏는다.
2. **화면이 장난감이 아니라 도구다** — 전업 투자자는 시간 단위로 본다.
3. **톤은 원본을 잇는 것이지 새로 만드는 게 아니다** — 기존 `index.html` 의 SF Pro · 검정 배경 · ⛩ 미신 티커와 어색하지 않아야 한다.
